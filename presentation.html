Контекст: V1 → V2

V1 (июнь 2025): Тревел-ассистент на Gen-T. Подбирает направления, ищет билеты/отели, составляет план досуга. Планирование ~2 мин. 5 млн бронирований, 51,2 млн клиентов экосистемы.
Моё предложение (V2): трансформировать в полноценного lifecycle-компаньона с глубокой интеграцией экосистемы Т-Банка, мультимодальным входом, антифрод-защитой и проактивным сопровождением на всех этапах поездки.

Экосистемное преимущество + удержание

Почему Т-Банк, а не Google/Kayak/ChatGPT: ни один зарубежный конкурент не может предложить бесшовную интеграцию бронирования со страховкой, валютой, маркетплейсом, подписками и инвестициями в одном диалоге. Booking ушёл, Airbnb недоступен, оплата иностранных сервисов затруднена.

Удержание

~50% путешественников уходят на Aviasales/Ostrovok. Lifecycle-UX (чек-лист, напоминания, такси) даёт причину вернуться. Повторные покупки: 30% → 40%.

Снижение OpEx

70% обращений в поддержку - типовые. Copilot закрывает до 50% автоматически. У Т-Банка уже несколько AI-ботов на Gen-T; тревел - следующий.

Сценарии Copilot (1-2)

Сценарий 1: «Вдохновляющий»

Вход: «Хочу куда-нибудь на море в мае, бюджет 80 000 на двоих»

Уточняющие вопросы: пляжный отдых или с экскурсиями? Безвиз? Дети? Прямые рейсы?

Результат: shortlist 3 направления (Турция / Сочи / Вьетнам) с ценой, перелётом, визой, погодой. Кнопки: «Подробнее» / «Забронировать» / «Покажи ещё»

Сценарий 2: «Видео из соцсети»

Вход: ссылка на Reels про Бали

Copilot: распознаёт видео → извлекает: Бали, Убуд, рисовые террасы → проверяет на фрод → «Это Бали, район Убуд. На сколько дней? Нужен ли отель рядом?»

Результат: план (перелёт + 3 варианта отелей), предупреждение о визе/страховке, антифрод-проверка

Сценарии Copilot (3-4)

Сценарий 3: «Помоги выбрать»

Вход: 5 рейсов Москва → Стамбул: «Какой лучше?»

Copilot: видит контекст. Уточняет: «Что важнее - цена, время или возможность отмены?» Сравнивает по: цена, время, пересадки, багаж, отмена

Результат: таблица-сравнение топ-3 с рекомендацией. Все цены из API

Сценарий 4: «Компаньон после покупки»

Вход: Таиланд через 2 недели: «Что подготовить?»

Copilot: «Летите из Москвы? Первый раз в Таиланде?»

Принцип: LLM - это «мозг» для понимания и общения. Все факты (цены, наличие, условия) берутся только из проверенных API.

Качество и запреты

Copilot не имеет права:

  1. Называть цену, если она не из Pricing API в текущей сессии
  2. Говорить «гарантирую», «обещаю», «точно будет»
  3. Утверждать наличие мест без проверки Availability API
  4. Озвучивать условия отмены «по памяти»
  5. Рекомендовать направления с визой без предупреждения

Как предотвращаем ошибки:

  1. Все факты из API в реальном времени
  2. Дисклеймер: «Цены актуальны на момент запроса»
  3. Если API недоступен: честный fallback + ссылка
  4. Валидация ответов на сервере (Fact Validator)
  5. Логирование всех ответов с источником для аудита

Поток запроса

  1. Пользователь отправляет запрос из Travel UI (текст, ссылка на видео, или кнопка «Сравнить»)
  2. API Gateway аутентифицирует, проверяет лимиты, маршрутизирует
  3. Intent Parser + Context Manager разбирает интент, обогащает контекстом (история, экран, lifecycle, профиль). Для видео вызывает Media Analyzer
  4. Copilot Core (Gen-T) формирует ответ, обращаясь к доменным сервисам и Ecosystem Router
  5. Fact Validator проверяет: все факты подтверждены? формулировки безопасны?
  6. Travel UI получает streaming-ответ: карточки, чек-листы, кнопки → бронирование

Вспомогательные блоки

#БлокЗачем нужен
1Intent Parser + Context ManagerПревращает текст/видео в структурированный запрос; хранит контекст диалога и lifecycle
2Fact ValidatorПроверяет каждый ответ: цены/условия подтверждены API, нет запрещённых слов, антифрод-флаги
3Ecosystem RouterОпределяет релевантные продукты экосистемы (страховка, валюта, eSIM, маркетплейс), подгружает данные

Правила «контроль фактов»

  1. Все цены только из Pricing & Availability API. Copilot не хранит и не генерирует цены. Каждая цена привязана к API-вызову с timestamp
  2. При таймауте или ошибке API: честный fallback. «Не удалось проверить цену, вот ссылка на поиск». Никогда не подставляет «примерную» цену
  3. Запрет на «обещания». Системный промпт запрещает «гарантирую», «обещаю», «точно», «100%». Fact Validator блокирует
  4. Условия тарифа: только пересказ, не интерпретация. Copilot резюмирует из структурированных данных
  5. Дисклеймер при любых ценах: «Актуально на [время]. Окончательные условия на экране бронирования». Логирование с источником
